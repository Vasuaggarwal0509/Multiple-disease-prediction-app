Inference Methodology

Each disease is handled by a dedicated model trained on its specialized dataset. This page documents the architecture and inference pipeline for every model in the system.

Image-Based Disease Models

Each image model receives a raw upload, applies architecture-specific preprocessing, and returns a single prediction with per-class probabilities.

Disease Architecture Input Size Preprocessing Output Classes
Eye Disease (OCT) MobileNet V3 (fine-tuned) 224 x 224 x 3 MobileNet V3 preprocess_input ([-1, 1] scaling) Softmax (4-way) CNV, DME, DRUSEN, NORMAL
Pneumonia (X-Ray) Custom CNN (5 Conv+Pool blocks) 300 x 300 x 3 Rescale by 1/255 Sigmoid (binary) Normal, Pneumonia
Malaria (Cell) Custom CNN (5 Conv+Pool blocks) 130 x 130 x 3 Rescale by 1/255 Sigmoid (binary) Parasitized, Uninfected
Inference Pipeline
  1. User uploads an image via the browser interface.
  2. Image is converted to RGB and resized to the target input dimensions for the specific model.
  3. Architecture-specific preprocessing is applied (either preprocess_input or simple 1/255 rescaling).
  4. The preprocessed tensor is passed to the trained model for inference.
  5. Output probabilities are returned alongside the predicted class label and confidence score.
Tabular Clinical Data Models

Clinical feature values are passed to trained Support Vector Machine classifiers. All tabular models use linear kernel SVM with StandardScaler preprocessing applied during training.
